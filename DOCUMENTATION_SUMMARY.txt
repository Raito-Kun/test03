================================================================================
KEY ARCHITECTURAL PATTERNS DOCUMENTED
================================================================================

1. Route → Controller → Service → Prisma
   - HTTP request → validation → business logic → database
   - Strict layer separation
   - Single responsibility per layer

2. Middleware Chain
   Auth → RBAC → Data Scope → Route Handler → Error Handler
   - Sequential security and filtering
   - Guaranteed user context
   - Automatic row-level filtering

3. Error Handling
   Service throws error with { code } property
   Controller catches and responds with appropriate status
   Global error handler catches unexpected exceptions
   Consistent response format

4. Data Scoping
   Admin/Manager/QA: See all data
   Leader: See team data
   Agents: See own data
   Middleware-enforced, service-applied
